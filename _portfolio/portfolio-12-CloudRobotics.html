rosconnect is an open-source ROS package that provides a framework to publish/subscribe ROS topics, call services, and fetch parameters via WAMP protocol between two ROS-based environments. It is a Cloud Robotics messaging framework that is responsible for secure and bi-directional data transmission between robots and a cloud server. The open-source third-party modules involved for the usage of this package are Crossbar.io and Rosbridge Protocol. One main advantage of using WAMP and Crossbar.io is their support for JavaScript. It means that data can be easily sent and received via a web browser and empower the IoT devices even more.

One of the use cases which is a clone-based architecture has been shown below. With this architecture, the implementation of a distributed and elastic computing environment would be possible which is also highly flexible in terms of the number of robots.



Configuration


The initial configuration is a JSON file located on the serverside. The configurations related to what type of data (topic, service, param) the client and server have to provide for each other is written inside this configuration file. When the client connects to the server, the initial configuration will be done and the connection will be fully established.
